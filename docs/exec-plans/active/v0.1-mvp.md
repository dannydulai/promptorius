# Execution Plan: v0.1 MVP

## Goal
Render a working prompt from config + Rhai scripts. Enough to `eval "$(promptorius init zsh)"` and get a functional prompt.

## Status: In Progress

## Completed
- [x] Project scaffold (Cargo.toml, module structure)
- [x] AGENTS.md, ARCHITECTURE.md, design docs
- [x] Config parsing and validation (config/mod.rs)
- [x] Rhai engine setup with string coercion (script/mod.rs)
- [x] Host API: environment (env, env_set, cwd, os)
- [x] Host API: filesystem (file_exists, read_file, glob_files, find_upward)
- [x] Host API: command execution (exec, exec_ok)
- [x] Host API: git (git_is_repo, git_branch, git_root, git_status)
- [x] Host API: color and icon
- [x] Host API: cache (cache_set, cache_get, cache_has, cache_del)
- [x] Host API: --cmd function registration and parsing
- [x] Host API: segment config map
- [x] Render module (color_to_ansi, parse_color, hex support)

- [x] Shell init scripts (zsh, bash, fish)
- [x] Stdlib scripts (directory, git, language, time, duration, exitcode, character, jobs, user_host)
- [x] Tests for config parsing, render, script coercion, cmd parsing

## Remaining
- [ ] **Pipeline**: wire up segment resolution, concurrent execution, format template evaluation
- [ ] **Format template parser**: extract `{expr}` blocks, handle `{{`/`}}` escaping
- [ ] **s() function**: register in Rhai, looks up segment output by name
- [ ] **Segment executor**: run scripts concurrently via rayon with global timeout
- [ ] **CLI dispatch**: wire `cli::run` to pipeline for default prompt rendering
- [ ] **Cache persistence**: load/save cache to XDG_CACHE_HOME between renders
- [ ] **Icon map**: load nerd font icon names -> unicode codepoints
- [ ] **Exec timeout**: enforce per-exec-call timeout (default 10ms)
- [ ] **Explain subcommand**: show segment timing and output
- [ ] **Check subcommand**: validate config and script syntax
- [ ] **Script new subcommand**: scaffold a new .rhai file
- [ ] **Completions subcommand**: generate shell completions via clap
- [ ] **Stdlib bundling**: embed stdlib scripts in binary or install alongside

## Decisions
- Using rayon for concurrency (not tokio) — prompt rendering is CPU-bound, not IO-bound
- Rhai AST disk caching deferred to post-MVP (in-memory caching per render is sufficient for now)
